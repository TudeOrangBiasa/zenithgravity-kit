# VERIFICATION GATE

## STRATEGY
1. Narrowest check first -> 2. Broad checks as needed.
- Isolate new failures from pre-existing issues.

## REPORT
- Executed checks.
- Outcome summary.
- Residual risks/unverified areas.

## DONT
- Fix unrelated tests unless requested.
